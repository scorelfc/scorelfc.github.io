D09: Zentrale Arbeitsprozesse finden auf der Plattform statt

(Einheit von Arbeitsumgebung und Produkt)

Arbeitsprozesse wie etwa die Produktion von Text, das Erstellen von Quellenverzeichnissen oder das Einpflegen von Datenbeständen finden für alle Beteiligten sichtbar auf der Plattform bzw. wo möglich direkt auf der Canvas statt. Es ist auch während der Arbeit an einzelnen Elementen jederzeit für alle beteiligten ersichtlich, auf welchem Stand Prozess und Produkt aktuell sind, Anpassungen der eigenen Tätigkeiten sind so auch on the fly möglich (G05 Same Page). So können Aktivitäten anderer direkt zur Kenntnis genommen werden (G03 Vital Signs), was die Identifikation darauf bezogener eigener Anschlussaktivitäten ermöglicht (G08 Productive Diversity, G09 Collective Agency). Zudem befördert dieses Prinzip die Produktion von kleineren Änderungen (Mikrobeiträgen), an denen weitere kleine Änderungen angeschlossen werden können. Das Einstellen von extern erarbeiteten umfangreichen Beiträgen (Codebombs), die für eine Anschlusshandeln zunächst eine umfassende Beschäftigung benötigen, wird so verhindert (G09 Collective Agency). Auch implizite Aushandlungsprozesse werden so befördert, etwa indem kleine Änderungen zeitnah begründet zurückgesetzt oder weiterentwickelt werden, ohne, dass dadurch größere Arbeitsergebnisse verworfen werden müssten. Hier ist eine Idee einer continuous innovation angelegt (G06 Joint Quality Care, G09 Collective Agency).
